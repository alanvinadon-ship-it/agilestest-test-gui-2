# Performance Testing Suite — Multi-Engine AI Routing\n\nAutomated load testing using Artillery to validate engine resolution performance under various load conditions.\n\n## Quick Start\n\n1. **Install dependencies**:\n   ```bash\n   pnpm add -D artillery\n   ```\n\n2. **Start the server**:\n   ```bash\n   npm run dev\n   ```\n\n3. **Run all tests**:\n   ```bash\n   ./run-tests.sh\n   ```\n\n## Test Scenarios\n\n| Test | Duration | Peak RPS | Concurrent Users | Purpose |\n|------|----------|----------|------------------|----------|\n| **Baseline** | 30s | 1 | 1 | Establish baseline performance |\n| **Stress** | 7m | 100 | 100 | Sustained load validation |\n| **Spike** | 4m | 150 | 150 | Spike resilience testing |\n\n## SLA Targets\n\n- **P50 (Median)**: < 30ms\n- **P95 (95th percentile)**: < 50ms\n- **P99 (99th percentile)**: < 100ms\n- **Error Rate**: < 0.1%\n\n## Files\n\n- `baseline.yml` — Baseline test configuration (1 RPS)\n- `stress.yml` — Stress test configuration (ramp to 100 RPS)\n- `spike.yml` — Spike test configuration (sudden jump to 150 RPS)\n- `processor.js` — Artillery processor for metrics and SLA validation\n- `run-tests.sh` — Automated test runner script\n- `results/` — Test results and HTML reports\n\n## Running Tests\n\n### All Tests\n```bash\n./run-tests.sh\n```\n\n### Individual Tests\n```bash\nartillery run baseline.yml\nartillery run stress.yml\nartillery run spike.yml\n```\n\n### With Output File\n```bash\nartillery run baseline.yml -o results/baseline.json\n```\n\n### Generate HTML Report\n```bash\nartillery report results/baseline.json\n```\n\n## Understanding Results\n\nSee `../docs/PERFORMANCE_TESTING_GUIDE.md` for detailed explanation of metrics, SLA validation, optimization strategies, and troubleshooting.\n\n## Key Metrics\n\n- **Latency**: Request response time (min, avg, p50, p95, p99, max)\n- **Throughput**: Requests per second (RPS)\n- **Success Rate**: Percentage of successful requests (HTTP 2xx)\n- **Error Rate**: Percentage of failed requests (HTTP 4xx, 5xx, timeout)\n- **Concurrency**: Number of simultaneous connections\n\n## SLA Violations\n\nThe processor monitors SLA compliance:\n- **P95 > 50ms**: Warning logged\n- **P99 > 100ms**: Violation logged\n\nViolations are printed to console and tracked in metrics.\n\n## CI/CD Integration\n\nAdd to GitHub Actions to run tests on every commit:\n\n```yaml\n- run: npm run dev &\n- run: sleep 5\n- run: npx artillery run load-tests/baseline.yml -o results/baseline.json\n- run: npx artillery report results/baseline.json\n```\n\n## Troubleshooting\n\n**Server not running**: Start with `npm run dev`\n\n**Artillery not found**: Install with `pnpm add -D artillery`\n\n**High latency**: Check database indexes, connection pool, and resource utilization\n\n**High error rate**: Check server logs, database connectivity, and timeout settings\n\nSee `../docs/PERFORMANCE_TESTING_GUIDE.md` for detailed troubleshooting.\n
