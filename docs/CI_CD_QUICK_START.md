# CI/CD Performance Testing — Quick Start\n\nGet performance testing running in your CI/CD pipeline in 5 minutes.\n\n## 1. Add GitHub Secrets (2 minutes)\n\n**Go to**: Repository Settings → Secrets and variables → Actions\n\n**Add these secrets**:\n\n| Secret | Value | Required |\n|--------|-------|----------|\n| `SLACK_WEBHOOK_URL` | Your Slack webhook URL | Optional |\n| `METRICS_DB_URL` | Database connection string | Optional |\n\n**Get Slack Webhook**:\n1. Go to https://api.slack.com/messaging/webhooks\n2. Create Incoming Webhook\n3. Copy URL and paste as secret\n\n## 2. Verify Workflow File (1 minute)\n\nThe workflow file is already created at:\n```\n.github/workflows/performance-tests.yml\n```\n\nNo changes needed — it's ready to use!\n\n## 3. Push Code to Trigger Tests (1 minute)\n\n```bash\ngit add .\ngit commit -m \"Enable performance testing in CI/CD\"\ngit push origin main\n```\n\n## 4. Monitor Results (1 minute)\n\n**View workflow runs**:\n1. Go to Actions tab in GitHub\n2. Click \"Performance Tests\"\n3. View latest run\n\n**View artifacts**:\n- Click on workflow run\n- Scroll to \"Artifacts\" section\n- Download `performance-reports`\n\n**View PR comments**:\n- Performance metrics auto-posted on PRs\n- Shows P50, P95, P99, success rate\n\n## 5. Setup Slack Notifications (Optional, 1 minute)\n\nIf you added `SLACK_WEBHOOK_URL` secret:\n- Workflow automatically posts to Slack on failures\n- Includes commit, branch, and link to details\n\n## What Happens Automatically\n\n✅ **On every push to main/develop**:\n- Runs baseline test (1 RPS)\n- Runs stress test (100 RPS)\n- Validates SLA compliance\n- Detects regressions\n- Generates HTML reports\n- Posts PR comments\n- Stores metrics\n- Alerts Slack (if configured)\n\n✅ **Daily at 2 AM UTC**:\n- Runs full performance test suite\n- Sends results to Slack\n- Stores historical metrics\n\n## Understanding Results\n\n### SLA Targets\n\n| Metric | Target | Status |\n|--------|--------|--------|\n| P50 (Median) | < 30ms | ✓ |\n| P95 (95th percentile) | < 50ms | ✓ |\n| P99 (99th percentile) | < 100ms | ✓ |\n| Error Rate | < 0.1% | ✓ |\n\n### Example PR Comment\n\n```\n## Performance Test Results\n\n### Baseline Test (1 RPS)\n- P50: 15ms ✓\n- P95: 40ms ✓\n- P99: 90ms ✓\n- Success Rate: 100.00%\n\n### Stress Test (100 RPS)\n- P50: 20ms ✓\n- P95: 48ms ✓\n- P99: 98ms ✓\n- Success Rate: 99.95%\n```\n\n## Troubleshooting\n\n### Workflow doesn't run\n\n**Check**:\n1. Is `.github/workflows/performance-tests.yml` committed?\n2. Are you pushing to `main` or `develop`?\n3. Is GitHub Actions enabled in settings?\n\n### Server fails to start\n\n**Check**:\n1. Does `npm run dev` work locally?\n2. Are environment variables set?\n3. Is database running?\n\n### No Slack notifications\n\n**Check**:\n1. Is `SLACK_WEBHOOK_URL` secret set?\n2. Is webhook URL valid?\n3. Check workflow logs for errors\n\n## Next Steps\n\n1. **Customize thresholds**: Edit `scripts/validate-sla.js` to adjust SLA targets\n2. **Add database storage**: Set `METRICS_DB_URL` to track metrics over time\n3. **Configure alerts**: Add custom notifications for your team\n4. **Review trends**: Analyze historical metrics to identify patterns\n\n## Full Documentation\n\nSee `CI_CD_PERFORMANCE_INTEGRATION.md` for:\n- Detailed workflow configuration\n- Advanced customization\n- Performance optimization strategies\n- Troubleshooting guide\n- Best practices\n\n## Support\n\nFor issues or questions:\n1. Check workflow logs in GitHub Actions\n2. Review `/tmp/server.log` for server errors\n3. See `CI_CD_PERFORMANCE_INTEGRATION.md` troubleshooting section\n
