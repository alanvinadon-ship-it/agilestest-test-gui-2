# Keycloak Authentication Migration Guide\n\nComplete guide for replacing Manus OAuth with Keycloak OpenID Connect (OIDC) authentication.\n\n## Overview\n\nThis guide walks through migrating from Manus OAuth to Keycloak OIDC. Keycloak provides:\n- Self-hosted identity management\n- OpenID Connect (OIDC) and OAuth 2.0 support\n- User federation and LDAP integration\n- Fine-grained access control\n- Session management and token refresh\n\n## Prerequisites\n\n1. **Keycloak Server**: Running Keycloak instance (v21+)\n2. **Node.js**: v20+\n3. **Dependencies**: `openid-client`, `jose` (already installed)\n\n## Step 1: Keycloak Server Setup\n\n### Docker Compose Setup\n\n```yaml\nversion: '3.8'\nservices:\n  keycloak:\n    image: quay.io/keycloak/keycloak:latest\n    environment:\n      KEYCLOAK_ADMIN: admin\n      KEYCLOAK_ADMIN_PASSWORD: admin123\n      KC_DB: mysql\n      KC_DB_URL: jdbc:mysql://mysql:3306/keycloak\n      KC_DB_USERNAME: keycloak\n      KC_DB_PASSWORD: keycloak123\n    ports:\n      - \"8080:8080\"\n    command:\n      - \"start-dev\"\n    depends_on:\n      - mysql\n\n  mysql:\n    image: mysql:8.0\n    environment:\n      MYSQL_ROOT_PASSWORD: root123\n      MYSQL_DATABASE: keycloak\n      MYSQL_USER: keycloak\n      MYSQL_PASSWORD: keycloak123\n    ports:\n      - \"3306:3306\"\n    volumes:\n      - mysql_data:/var/lib/mysql\n\nvolumes:\n  mysql_data:\n```\n\n### Manual Setup\n\n1. Download Keycloak: https://www.keycloak.org/downloads\n2. Extract and run:\n   ```bash\n   bin/kc.sh start-dev\n   ```\n3. Access admin console: http://localhost:8080/admin\n4. Login with admin/admin\n\n## Step 2: Create Keycloak Realm and Client\n\n### Create Realm\n\n1. Click \"Master\" dropdown → \"Create realm\"\n2. Name: `agilestest`\n3. Click \"Create\"\n\n### Create Client\n\n1. Go to \"Clients\" → \"Create client\"\n2. Client ID: `agilestest-web`\n3. Click \"Next\"\n4. Enable \"Client authentication\"\n5. Click \"Next\" → \"Save\"\n\n### Configure Client\n\n**General Settings**:\n- Access Type: `confidential`\n- Standard Flow Enabled: ✓\n- Direct Access Grants Enabled: ✓\n- Valid Redirect URIs: `http://localhost:3000/api/oauth/callback`\n- Web Origins: `http://localhost:3000`\n\n**Credentials Tab**:\n- Copy `Client Secret` (you'll need this)\n\n**Mapper**:\n- Add mapper → User Property\n- Property: `email`\n- Token Claim Name: `email`\n\n## Step 3: Environment Variables\n\nAdd to `.env`:\n\n```bash\n# Keycloak Configuration\nKEYCLOAK_URL=http://localhost:8080\nKEYCLOAK_REALM=agilestest\nKEYCLOAK_CLIENT_ID=agilestest-web\nKEYCLOAK_CLIENT_SECRET=your-client-secret-from-step-2\nKEYCLOAK_REDIRECT_URI=http://localhost:3000/api/oauth/callback\n\n# Keep existing\nJWT_SECRET=your-jwt-secret\nDATABASE_URL=mysql://user:password@localhost:3306/agilestest\n```\n\n## Step 4: Update Backend OAuth Routes\n\n### Replace Manus OAuth with Keycloak\n\n**File**: `server/_core/index.ts`\n\n```typescript\n// Replace this:\nimport { registerOAuthRoutes } from \"./oauth\";\n// With this:\nimport { registerKeycloakOAuthRoutes } from \"./keycloak-oauth\";\n\n// In app setup:\n// Replace:\nregisterOAuthRoutes(app);\n// With:\nregisterKeycloakOAuthRoutes(app);\n```\n\n### Create Keycloak OAuth Module\n\n**File**: `server/_core/keycloak-oauth.ts`\n\n```typescript\nimport type { Express, Request, Response } from \"express\";\nimport { Issuer } from \"openid-client\";\nimport { SignJWT, jwtVerify } from \"jose\";\nimport * as db from \"../db\";\nimport { getSessionCookieOptions } from \"./cookies\";\n\nconst COOKIE_NAME = \"session\";\nconst ONE_YEAR_MS = 365 * 24 * 60 * 60 * 1000;\n\nclass KeycloakOAuthService {\n  private client: any = null;\n  private initialized = false;\n\n  async initialize(): Promise<void> {\n    if (this.initialized) return;\n\n    const keycloakUrl = process.env.KEYCLOAK_URL;\n    const realm = process.env.KEYCLOAK_REALM;\n    const clientId = process.env.KEYCLOAK_CLIENT_ID;\n    const clientSecret = process.env.KEYCLOAK_CLIENT_SECRET;\n\n    if (!keycloakUrl || !realm || !clientId || !clientSecret) {\n      throw new Error(\"Missing Keycloak configuration\");\n    }\n\n    const issuerUrl = `${keycloakUrl}/realms/${realm}`;\n    console.log(`[Keycloak] Discovering issuer: ${issuerUrl}`);\n\n    const issuer = await Issuer.discover(issuerUrl);\n    console.log(`[Keycloak] Issuer discovered`);\n\n    this.client = new issuer.Client({\n      client_id: clientId,\n      client_secret: clientSecret,\n      redirect_uris: [process.env.KEYCLOAK_REDIRECT_URI!],\n      response_types: [\"code\"],\n    });\n\n    this.initialized = true;\n  }\n\n  getAuthorizationUrl(state: string): string {\n    return this.client.authorizationUrl({\n      scope: \"openid profile email\",\n      state,\n    });\n  }\n\n  async exchangeCodeForToken(code: string): Promise<any> {\n    return await this.client.callback(\n      process.env.KEYCLOAK_REDIRECT_URI!,\n      { code }\n    );\n  }\n\n  async getUserInfo(accessToken: string): Promise<any> {\n    return await this.client.userinfo(accessToken);\n  }\n}\n\nconst keycloakService = new KeycloakOAuthService();\n\nasync function createSessionToken(\n  openId: string,\n  name: string\n): Promise<string> {\n  const secret = new TextEncoder().encode(process.env.JWT_SECRET!);\n\n  return await new SignJWT({\n    openId,\n    name,\n    appId: process.env.VITE_APP_ID,\n  })\n    .setProtectedHeader({ alg: \"HS256\" })\n    .setExpirationTime(Math.floor(Date.now() / 1000) + ONE_YEAR_MS / 1000)\n    .sign(secret);\n}\n\nexport function registerKeycloakOAuthRoutes(app: Express) {\n  // Initialize on first request\n  app.use(async (req, res, next) => {\n    try {\n      await keycloakService.initialize();\n      next();\n    } catch (error) {\n      console.error(\"[Keycloak] Init error:\", error);\n      res.status(500).json({ error: \"Auth service unavailable\" });\n    }\n  });\n\n  // Login route\n  app.get(\"/api/oauth/login\", (req: Request, res: Response) => {\n    try {\n      const returnTo = (req.query.return_to as string) || \"/\";\n      const state = Buffer.from(returnTo).toString(\"base64\");\n      const authUrl = keycloakService.getAuthorizationUrl(state);\n      res.redirect(authUrl);\n    } catch (error) {\n      res.status(500).json({ error: \"Login failed\" });\n    }\n  });\n\n  // Callback route\n  app.get(\"/api/oauth/callback\", async (req: Request, res: Response) => {\n    const code = req.query.code as string;\n    const state = req.query.state as string;\n\n    if (!code || !state) {\n      res.status(400).json({ error: \"Missing code or state\" });\n      return;\n    }\n\n    try {\n      const tokenSet = await keycloakService.exchangeCodeForToken(code);\n      const userInfo = await keycloakService.getUserInfo(tokenSet.access_token);\n\n      await db.upsertUser({\n        openId: userInfo.sub,\n        name: userInfo.name || null,\n        email: userInfo.email || null,\n        loginMethod: \"keycloak\",\n        lastSignedIn: new Date(),\n      });\n\n      const sessionToken = await createSessionToken(\n        userInfo.sub,\n        userInfo.name || \"\"\n      );\n\n      const cookieOptions = getSessionCookieOptions(req);\n      res.cookie(COOKIE_NAME, sessionToken, {\n        ...cookieOptions,\n        maxAge: ONE_YEAR_MS,\n      });\n\n      const redirectTo = Buffer.from(state, \"base64\").toString();\n      res.redirect(302, redirectTo);\n    } catch (error) {\n      console.error(\"[Keycloak] Callback error:\", error);\n      res.status(500).json({ error: \"Callback failed\" });\n    }\n  });\n\n  // Logout route\n  app.get(\"/api/oauth/logout\", (req: Request, res: Response) => {\n    res.clearCookie(COOKIE_NAME);\n    res.json({ success: true });\n  });\n}\n```\n\n## Step 5: Update Frontend Login URL\n\n**File**: `client/src/const.ts`\n\n```typescript\nexport function getLoginUrl(returnPath?: string): string {\n  const origin = typeof window !== \"undefined\" ? window.location.origin : \"\";\n  const returnTo = returnPath ? `${origin}${returnPath}` : origin;\n  const encoded = Buffer.from(returnTo).toString(\"base64\");\n  return `/api/oauth/login?return_to=${encoded}`;\n}\n```\n\n## Step 6: Update Frontend Auth Hook\n\n**File**: `client/src/_core/hooks/useAuth.ts`\n\nNo changes needed - the hook already uses `/api/oauth/login` and `/api/oauth/logout`.\n\n## Step 7: Database Migration\n\nUpdate NULL;\n```\n\n## Step 8: Testing\n\n### Test Login Flow\n\n1. Set up user federation if needed\n\n## Troubleshooting\n\n### \"Issuer discovery failed\"\n\n**Cause**: Keycloak URL is incorrect or server is down\n\n**Solution**:\n1. Verify Keycloak is running: `curl http://localhost:8080`\n2. Check `KEYCLOAK_URL` environment variable\n3. Verify realm exists: `curl http://localhost:8080/realms/agilestest`\n\n### \"Invalid client credentials\"\n\n**Cause**: Client ID or secret is wrong\n\n**Solution**:\n1. Go to Keycloak admin console\n2. Verify client ID matches `KEYCLOAK_CLIENT_ID`\n3. Copy correct secret from Credentials tab\n\n### \"Redirect URI mismatch\"\n\n**Cause**: Callback URL doesn't match Keycloak configuration\n\n**Solution**:\n1. Go to Keycloak client settings\n2. Add `http://localhost:3000/api/oauth/callback` to Valid Redirect URIs\n3. Verify `KEYCLOAK_REDIRECT_URI` matches exactly\n\n### User not created\n\n**Cause**: Database upsert failed\n\n**Solution**:\n1. Check database connection\n2. Verify `users` table exists\n3. Check application logs for errors\n\n## Comparison: Manus vs Keycloak\n\n| Feature | Manus | Keycloak |\n|---------|-------|----------|\n| Hosting | Cloud | Self-hosted |\n| Setup | Built-in | Manual setup |\n| Cost | Included | Free (self-hosted) |\n| Customization | Limited | Extensive |\n| User Management | Basic | Advanced |\n| LDAP/Federation | No | Yes |\n| Multi-tenancy | No | Yes |\n| Token Refresh | Automatic | Manual |\n\n## Next Steps\n\n1. **User Federation**: Connect LDAP/Active Directory\n2. **Social Login**: Add Google, GitHub, etc.\n3. **Custom Themes**: Customize login page\n4. **Role-Based Access**: Implement RBAC with Keycloak roles\n5. **Audit Logging**: Enable event logging\n\n## References\n\n- [Keycloak Documentation](https://www.keycloak.org/documentation)\n- [OpenID Connect Spec](https://openid.net/connect/)\n- [openid-client Library](https://github.com/panva/node-openid-client)\n- [JWT Best Practices](https://tools.ietf.org/html/rfc8725)\n
