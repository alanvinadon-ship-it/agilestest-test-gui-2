# Social Login API Reference\n\nAPI endpoints and integration guide for social login functionality.\n\n## Authentication Flow\n\n### Standard OAuth 2.0 Authorization Code Flow\n\n```\n┌─────────┐                                      ┌──────────────┐\n│ Browser │                                      │ Keycloak     │\n└────┬────┘                                      └──────┬───────┘\n     │                                                   │\n     │ 1. GET /api/oauth/login?return_to=/dashboard    │\n     ├──────────────────────────────────────────────────>│\n     │                                                   │\n     │ 2. Redirect to Google/GitHub                     │\n     │<──────────────────────────────────────────────────┤\n     │                                                   │\n     │ 3. User authorizes                              │\n     │ (on Google/GitHub)                              │\n     │                                                   │\n     │ 4. Redirect to callback with code               │\n     │<──────────────────────────────────────────────────┤\n     │                                                   │\n     │ 5. GET /api/oauth/callback?code=...&state=...   │\n     ├──────────────────────────────────────────────────>│\n     │                                                   │\n     │ 6. Redirect to /dashboard with session cookie   │\n     │<──────────────────────────────────────────────────┤\n     │                                                   │\n```\n\n## API Endpoints\n\n### GET /api/oauth/login\n\nInitiate login flow.\n\n**Parameters**:\n```\nreturn_to: string (optional)\n  - URL to redirect to after login\n  - Default: /\n  - Example: /dashboard\n```\n\n**Example**:\n```bash\ncurl -X GET \"http://localhost:3000/api/oauth/login?return_to=/dashboard\"\n```\n\n**Response**:\n- Redirect to Keycloak login page\n- Keycloak displays Google and GitHub buttons\n\n### GET /api/oauth/callback\n\nHandle OAuth provider callback.\n\n**Parameters** (from provider):\n```\ncode: string\n  - Authorization code from provider\n  - Required\n\nstate: string\n  - State parameter for CSRF protection\n  - Required\n```\n\n**Example** (automatic, called by provider):\n```\nGET http://localhost:3000/api/oauth/callback?code=abc123&state=xyz789\n```\n\n**Response**:\n- Sets session cookie\n- Redirects to return_to URL (or /)\n- User is now authenticated\n\n### GET /api/oauth/logout\n\nLogout user.\n\n**Example**:\n```bash\ncurl -X GET \"http://localhost:3000/api/oauth/logout\"\n```\n\n**Response**:\n```json\n{\n  \"success\": true\n}\n```\n\n## User Data\n\n### User Object\n\nAfter successful login, user data is available:\n\n```typescript\ninterface User {\n  openId: string;           // Unique identifier\n  name: string | null;      // User's name\n  email: string | null;     // User's email\n  loginMethod: string;      // \"google\" or \"github\"\n  lastSignedIn: Date;       // Last login timestamp\n}\n```\n\n### Get Current User\n\n**Endpoint**: `GET /api/trpc/auth.me`\n\n**Example**:\n```bash\ncurl -X GET \"http://localhost:3000/api/trpc/auth.me\" \\\n  -H \"Cookie: session=YOUR_SESSION_TOKEN\"\n```\n\n**Response**:\n```json\n{\n  \"result\": {\n    \"data\": {\n      \"openId\": \"google-118234567890\",\n      \"name\": \"John Doe\",\n      \"email\": \"john@example.com\",\n      \"loginMethod\": \"google\",\n      \"lastSignedIn\": \"2026-03-01T12:00:00Z\"\n    }\n  }\n}\n```\n\n## Frontend Integration\n\n### Login Button\n\n```typescript\nimport { getLoginUrl } from '@/const';\n\nfunction LoginButton() {\n  return (\n    <a href={getLoginUrl('/dashboard')}>\n      Login\n    </a>\n  );\n}\n```\n\n### Logout Button\n\n```typescript\nfunction LogoutButton() {\n  const handleLogout = async () => {\n    await fetch('/api/oauth/logout');\n    window.location.href = '/';\n  };\n\n  return (\n    <button onClick={handleLogout}>\n      Logout\n    </button>\n  );\n}\n```\n\n### Protected Route\n\n```typescript\nimport { useAuth } from '@/_core/hooks/useAuth';\n\nfunction ProtectedPage() {\n  const { user, loading, isAuthenticated } = useAuth();\n\n  if (loading) return <div>Loading...</div>;\n  if (!isAuthenticated) return <div>Not authenticated</div>;\n\n  return (\n    <div>\n      <h1>Welcome, {user?.name}</h1>\n      <p>Email: {user?.email}</p>\n      <p>Provider: {user?.loginMethod}</p>\n    </div>\n  );\n}\n```\n\n## Session Management\n\n### Session Cookie\n\n**Name**: `session`\n\n**Properties**:\n- HttpOnly: true (not accessible from JavaScript)\n- Secure: true (HTTPS only in production)\n- SameSite: Lax\n- Max-Age: 1 year\n\n### Token Refresh\n\nTokens are automatically refreshed:\n- Session cookie is renewed on each request\n- No manual refresh needed\n- Tokens expire after 1 year\n\n### Logout\n\nLogout clears the session cookie:\n```bash\nGET /api/oauth/logout\n```\n\n## Error Handling\n\n### Common Errors\n\n**Missing code or state**\n```json\n{\n  \"error\": \"code and state are required\"\n}\n```\n\n**Missing openId from user info**\n```json\n{\n  \"error\": \"openId missing from user info\"\n}\n```\n\n**OAuth callback failed**\n```json\n{\n  \"error\": \"OAuth callback failed\"\n}\n```\n\n### Error Handling in Frontend\n\n```typescript\ntry {\n  const response = await fetch('/api/oauth/callback?code=...&state=...');\n  if (!response.ok) {\n    const error = await response.json();\n    console.error('Login failed:', error.error);\n  }\n} catch (error) {\n  console.error('Login error:', error);\n}\n```\n\n## Security Considerations\n\n### CSRF Protection\n\n- State parameter is used to prevent CSRF attacks\n- State is base64-encoded return URL\n- Validated on callback\n\n### Token Security\n\n- JWT tokens signed with HS256\n- Signed with JWT_SECRET environment variable\n- Tokens contain: openId, name, appId\n- Expiration: 1 year\n\n### Secure Redirect\n\n- Return URL is base64-encoded in state\n- Prevents open redirect attacks\n- Only redirects to same origin\n\n## Rate Limiting\n\nNo built-in rate limiting. Implement at reverse proxy:\n\n```nginx\nlimit_req_zone $binary_remote_addr zone=login:10m rate=5r/m;\nlimit_req /api/oauth/login zone=login burst=10 nodelay;\n```\n\n## Monitoring\n\n### Log Login Events\n\nEnable Keycloak event logging:\n\n1. Keycloak Admin → Realm Settings → Events\n2. Enable \"Save Events\"\n3. View in \"Events\" tab\n\n### Metrics\n\nTrack:\n- Login attempts\n- Login failures\n- Provider usage (Google vs GitHub)\n- Session duration\n- Active users\n\n## Testing\n\n### Manual Testing\n\n```bash\n# 1. Start Keycloak\ndocker-compose -f docker-compose.keycloak.yml up\n\n# 2. Start app\nnpm run dev\n\n# 3. Test login\nopen http://localhost:3000/api/oauth/login\n\n# 4. Authorize with Google/GitHub\n# 5. Should be redirected back\n\n# 6. Test logout\nopen http://localhost:3000/api/oauth/logout\n```\n\n### Automated Testing\n\n```typescript\n// Example test\ndescribe('Social Login', () => {\n  it('should login with Google', async () => {\n    const response = await fetch('/api/oauth/login');\n    expect(response.status).toBe(302);\n    expect(response.headers.get('location')).toContain('keycloak');\n  });\n\n  it('should logout', async () => {\n    const response = await fetch('/api/oauth/logout');\n    const data = await response.json();\n    expect(data.success).toBe(true);\n  });\n});\n```\n\n## Troubleshooting\n\n### User not created\n\n1. Check database connection\n2. Verify users table exists\n3. Check application logs\n4. Verify user info from provider\n\n### Session not persisting\n\n1. Check cookie settings\n2. Verify HttpOnly flag\n3. Check browser cookie policy\n4. Clear browser cookies and retry\n\n### Redirect loop\n\n1. Check return_to parameter\n2. Verify redirect URL is valid\n3. Check Keycloak configuration\n4. Clear browser cache\n\n## References\n\n- [OAuth 2.0 RFC 6749](https://tools.ietf.org/html/rfc6749)\n- [OpenID Connect Core](https://openid.net/specs/openid-connect-core-1_0.html)\n- [JWT Best Practices](https://tools.ietf.org/html/rfc8725)\n- [OWASP OAuth 2.0 Security](https://cheatsheetseries.owasp.org/cheatsheets/OAuth_2_0_Cheat_Sheet.html)\n
