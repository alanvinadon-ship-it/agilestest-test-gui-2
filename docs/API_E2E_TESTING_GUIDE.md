# API E2E Testing Guide — Multi-Engine AI Routing\n\nComprehensive guide for implementing 27 end-to-end test scenarios for the multi-engine AI routing system using Vitest and test utilities.\n\n## Overview\n\nThis guide covers automated testing of the Drive AI analysis feature with multi-engine routing. All 27 scenarios are organized into 7 test groups, testing engine resolution, routing rules, fallback behavior, result storage, multi-organization isolation, performance, and production scenarios.\n\n## Test Infrastructure\n\n### Available Test Utilities\n\nThe `server/test-utils.ts` module provides helpers for setting up test data:\n\n```typescript\n// Organization and user management\ncreateTestOrgId(): string\ncreateTestUserId(): string\n\n// Engine setup\ncreateTestEngines(orgId, userId): Promise<{ primaryEngineUid, secondaryEngineUid, tertiaryEngineUid }>\n\n// Routing rules\ncreateTestRoutingRule(orgId, userId, options): Promise<string>\n\n// Drive analysis\ncreateTestDriveRun(orgId, userId, projectUid?): Promise<string>\ncreateTestAnalysis(orgId, runUid, userId, mode?): Promise<string>\ncreateTestSegments(orgId, analysisUid, count?): Promise<string[]>\ncreateTestFeedback(orgId, analysisUid, score?, notes?, userId?): Promise<string>\n\n// Cleanup\ncleanupTestData(orgId): Promise<void>\n\n// Utilities\nmeasureTime<T>(fn): Promise<{ result: T, duration: number }>\n```\n\n## Test Scenarios (27 Total)\n\n### Group 1: Engine Resolution pattern:\n\n```typescript\nimport { describe, it, expect, beforeEach, afterEach } from \"vitest\";\nimport { createTestOrgId, createTestUserId, createTestEngines, cleanupTestData } from \"./test-utils\";\nimport { getDb } from \"./db\";\nimport { eq } from \"drizzle-orm\";\nimport { aiEngines } from \"../drizzle/schema\";\n\ndescribe(\"API E2E: Scenario Group\", () => {\n  let testOrgId: string;\n  let testUserId: string;\n  let primaryEngineUid: string;\n\n  beforeEach(async () => {\n    testOrgId = createTestOrgId();\n    testUserId = createTestUserId();\n    const engines = await createTestEngines(testOrgId, testUserId);\n    primaryEngineUid = engines.primaryEngineUid;\n  });\n\n  afterEach(async () => {\n    await cleanupTestData(testOrgId);\n  });\n\n  it(\"should test scenario\", async () => {\n    const db = await getDb();\n    if (!db) throw new Error(\"Database not available\");\n\n    // Test implementation\n    const [engine] = await db.select().from(aiEngines)\n      .where(eq(aiEngines.uid, primaryEngineUid)).limit(1);\n\n    expect(engine).toBeDefined();\n  });\n});\n```\n\n## Running Tests\n\n```bash\n# Run all tests\npnpm test\n\n# Run specific test file\npnpm test server/api-e2e-engines.test.ts\n\n# Run with coverage\npnpm test --coverage\n\n# Watch mode\npnpm test --watch\n```\n\n## Key Assertions\n\n- **Existence**: `expect(result).toBeDefined()` / `expect(result).toHaveLength(n)`\n- **Equality**: `expect(field).toBe(expectedValue)`\n- **Arrays**: `expect(array.map(x => x.field)).toEqual([...])`\n- **Performance**: `expect(duration).toBeLessThan(100)`\n- **Isolation**: `expect(orgAData).not.toEqual(orgBData)`\n\n## Database Query Patterns\n\n### Select Single Record\n```typescript\nconst [record] = await db.select().from(table)\n  .where(eq(table.uid, uid)).limit(1);\n```\n\n### Select Multiple Records\n```typescript\nconst records = await db.select().from(table)\n  .where(eq(table.orgId, orgId));\n```\n\n### Select with Conditions\n```typescript\nconst records = await db.select().from(table)\n  .where(and(\n    eq(table.orgId, orgId),\n    eq(table.enabled, true),\n    eq(table.useCase, \"DRIVE_DIAG\")\n  ));\n```\n\n### Select with Ordering\n```typescript\nconst records = await db.select().from(table)\n  .where(eq(table.orgId, orgId))\n  .orderBy(t => t.priority);\n```\n\n### Update Record\n```typescript\nawait db.update(table)\n  .set({ field: newValue })\n  .where(eq(table.uid, uid));\n```\n\n## Cleanup Strategy\n\nAlways call
