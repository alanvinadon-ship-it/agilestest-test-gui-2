# Keycloak Social Login Setup Guide\n\nComplete guide for enabling Google and GitHub social login in Keycloak.\n\n## Overview\n\nThis guide covers:\n- Setting up Google OAuth 2.0 credentials\n- Setting up GitHub OAuth application\n- Configuring Keycloak identity providers\n- Testing social login flows\n- Troubleshooting common issues\n\n## Part 1: Google OAuth Setup\n\n### Step 1: Create Google Cloud Project\n\n1. Go to [Google Cloud Console](https://console.cloud.google.com/)\n2. Click \"Select a Project\" → \"New Project\"\n3. Project name: `AgilesTest`\n4. Click \"Create\"\n5. Wait for project creation (2-3 minutes)\n\n### Step 2: Enable Google+ API\n\n1. In Google Cloud Console, go to \"APIs & Services\" → \"Library\"\n2. Search for \"Google+ API\"\n3. Click on it → \"Enable\"\n4. Wait for API to enable\n\n### Step 3: Create OAuth 2.0 Credentials\n\n1. Go to \"APIs & Services\" → \"Credentials\"\n2. Click \"Create Credentials\" → \"OAuth client ID\"\n3. If prompted, configure OAuth consent screen first:\n   - User Type: External\n   - Click \"Create\"\n   - Fill in app name: `AgilesTest`\n   - User support email: your-email@example.com\n   - Developer contact: your-email@example.com\n   - Click \"Save and Continue\"\n   - Skip scopes (use defaults)\n   - Skip test users\n   - Click \"Back to Dashboard\"\n\n4. Now create OAuth client ID:\n   - Application type: \"Web application\"\n   - Name: `AgilesTest Web Client`\n   - Authorized JavaScript origins:\n     ```\n     http://localhost:8080\n     http://localhost:3000\n     https://yourdomain.com\n     ```\n   - Authorized redirect URIs:\n     ```\n     http://localhost:8080/realms/agilestest/broker/google/endpoint\n     http://localhost:3000/api/oauth/callback\n     https://yourdomain.com/realms/agilestest/broker/google/endpoint\n     ```\n   - Click \"Create\"\n\n5. Copy the credentials:\n   - Client ID: `YOUR_GOOGLE_CLIENT_ID`\n   - Client Secret: `YOUR_GOOGLE_CLIENT_SECRET`\n\n### Step 4: Configure Google in Keycloak\n\n1. Go to Keycloak Admin Console: http://localhost:8080/admin\n2. Select realm: `agilestest`\n3. Go to \"Identity Providers\"\n4. Click \"Create\" → Select \"Google\"\n5. Fill in:\n   - Alias: `google`\n   - Display Name: `Google`\n   - Client ID: `YOUR_GOOGLE_CLIENT_ID`\n   - Client Secret: `YOUR_GOOGLE_CLIENT_SECRET`\n   - Default Scopes: `openid profile email`\n6. Click \"Save\"\n\n### Step 5: Configure Mapper (Optional)\n\n1. In Google identity provider settings, go to \"Mappers\"\n2. Click \"Create\"\n3. Name: `Email to email`\n4. Mapper Type: `Attribute Importer`\n5. Social Profile JSON Field Name: `email`\n6. User Attribute Name: `email`\n7. Click \"Save\"\n\n## Part 2: GitHub OAuth Setup\n\n### Step 1: Register GitHub Application\n\n1. Go to GitHub Settings: https://github.com/settings/developers\n2. Click \"New OAuth App\"\n3. Fill in:\n   - Application name: `AgilesTest`\n   - Homepage URL: `http://localhost:3000`\n   - Authorization callback URL:\n     ```\n     http://localhost:8080/realms/agilestest/broker/github/endpoint\n     ```\n4. Click \"Register application\"\n5. Copy credentials:\n   - Client ID: `YOUR_GITHUB_CLIENT_ID`\n   - Client Secret: `YOUR_GITHUB_CLIENT_SECRET` (click \"Generate a new client secret\")\n\n### Step 2: Configure GitHub in Keycloak\n\n1. Go to Keycloak Admin Console\n2. Select realm: `agilestest`\n3. Go to \"Identity Providers\"\n4. Click \"Create\" → Select \"GitHub\"\n5. Fill in:\n   - Alias: `github`\n   - Display Name: `GitHub`\n   - Client ID: `YOUR_GITHUB_CLIENT_ID`\n   - Client Secret: `YOUR_GITHUB_CLIENT_SECRET`\n   - Default Scopes: `user:email`\n6. Click \"Save\"\n\n### Step 3: Configure GitHub Mapper\n\n1. In GitHub identity provider settings, go to \"Mappers\"\n2. Click \"Create\"\n3. Name: `GitHub Email Mapper`\n4. Mapper Type: `Attribute Importer`\n5. Social Profile JSON Field Name: `email`\n6. User Attribute Name: `email`\n7. Click \"Save\"\n\n## Part 3: Keycloak Login Page Configuration\n\n### Enable Social Login Buttons\n\n1. Go to Keycloak realm settings\n2. Go to \"Themes\"\n3. Login Theme: `keycloak` (default)\n4. Account Theme: `keycloak`\n5. Admin Console Theme: `keycloak`\n6. Click \"Save\"\n\n### Customize Login Page (Optional)\n\n1. Go to \"Realm Settings\" → \"Themes\"\n2. Create custom theme or use default\n3. Social login buttons will appear automatically\n\n## Part 4: Frontend Integration\n\n### Update Login Component\n\nThe social login buttons appear automatically on the Keycloak login page. No frontend changes needed.\n\n**Login Flow**:\n1. User clicks \"Login\" on your app\n2. Redirected to Keycloak login page\n3. Keycloak shows \"Google\" and \"GitHub\" buttons\n4. User clicks social provider button\n5. Redirected to Google/GitHub login\n6. After authorization, redirected back to Keycloak\n7. Keycloak creates/links user account\n8. Redirected back to your app with session\n\n### User Account Linking (Optional)\n\nAllow users to link multiple social accounts:\n\n1. In Keycloak account console\n2. Go to \"Linked Accounts\"\n3. Users can link/unlink Google and GitHub\n\n## Part 5: Testing\n\n### Test Google Login\n\n1. Start Keycloak: `docker-compose up`\n2. Start your app: `npm run dev`\n3. Go to http://localhost:3000\n4. Click \"Login\"\n5. On Keycloak login page, click \"Google\"\n6. Login with Google account\n7. Authorize app access\n8. Should be redirected back to your app\n9. User should be logged in\n\n### Test GitHub Login\n\n1. Go to http://localhost:3000\n2. Click \"Login\"\n3. On Keycloak login page, click \"GitHub\"\n4. Login with GitHub account\n5. Authorize app access\n6. Should be redirected back to your app\n7. User should be logged in\n\n### Verify User Creation\n\n```bash\n# Check database for new user\nSELECT * FROM users WHERE login_method IN ('google', 'github');\n```\n\n## Part 6: Production Configuration\n\n### Update Redirect URIs\n\n**Google Cloud Console**:\n```\nAuthorized redirect URIs:\n- https://yourdomain.com/realms/agilestest/broker/google/endpoint\n- https://yourdomain.com/api/oauth/callback\n```\n\n**GitHub Settings**:\n```\nAuthorization callback URL:\n- https://yourdomain.com/realms/agilestest/broker/github/endpoint\n```\n\n**Keycloak Identity Providers**:\n- Update redirect URIs in each provider settings\n\n### Environment Variables\n\n```bash\n# .env.production\nKEYCLOAK_URL=https://keycloak.yourdomain.com\nKEYCLOAK_REALM=agilestest\nKEYCLOAK_CLIENT_ID=agilestest-web\nKEYCLOAK_CLIENT_SECRET=your-production-secret\nKEYCLOAK_REDIRECT_URI=https://app.yourdomain.com/api/oauth/callback\n```\n\n## Part 7: Advanced Configuration\n\n### User Attribute Mapping\n\nMap social provider attributes to Keycloak user attributes:\n\n1. Go to Identity Provider → Mappers\n2. Create mapper for each attribute:\n   - Email\n   - Name\n   - Picture\n   - Location\n   - etc.\n\n### Account Linking Strategy\n\n**Option 1: Automatic Linking** (Default)\n- Users with same email are automatically linked\n- Configured in Identity Provider → \"Link only existing accounts\"\n\n**Option 2: Manual Linking**\n- Users manually link accounts in account console\n- More secure, requires user action\n\n**Option 3: First-Come-First-Served**\n- First social login with email creates account\n- Subsequent logins with same email link to account\n\n### First Login Flow\n\nCustomize what happens on first social login:\n\n1. Go to Identity Provider settings\n2. \"First Login Flow\": Select flow\n3. Options:\n   - `browser` - Standard login flow\n   - `first broker login` - Custom first-time flow\n   - `registration` - Force registration\n\n## Part 8: Troubleshooting\n\n### \"Invalid redirect_uri\"\n\n**Cause**: Redirect URI doesn't match provider configuration\n\n**Solution**:\n1. Check Google Cloud Console authorized URIs\n2. Check GitHub OAuth app settings\n3. Ensure URIs match exactly (including protocol and trailing slash)\n\n### \"Client authentication failed\"\n\n**Cause**: Client ID or secret is incorrect\n\n**Solution**:\n1. Verify credentials in Google Cloud Console\n2. Verify credentials in GitHub settings\n3. Check Keycloak identity provider settings\n4. Regenerate credentials if needed\n\n### \"Email already exists\"\n\n**Cause**: Email from social provider already exists in database\n\n**Solution**:\n1. Option 1: Delete existing user and retry\n2. Option 2: Link accounts manually in account console\n3. Option 3: Use different email for social provider\n\n### User not created\n\n**Cause**: Database or Keycloak issue\n\n**Solution**:\n1. Check Keycloak logs: `docker logs keycloak`\n2. Check application logs\n3. Verify database connection\n4. Check user mapper configuration\n\n### Social buttons not appearing\n\n**Cause**: Identity providers not enabled or theme issue\n\n**Solution**:\n1. Verify identity providers are enabled in Keycloak\n2. Check realm theme is set correctly\n3. Clear browser cache\n4. Restart Keycloak\n\n## Part 9: Security Best Practices\n\n1. **Use HTTPS**: Always use HTTPS in production\n2. **Secure Secrets**: Store client secrets in environment variables\n3. **Limit Scopes**: Only request necessary scopes\n4. **Validate Tokens**: Verify JWT signatures\n5. **Monitor Logins**: Log all authentication events\n6. **Rate Limiting**: Implement rate limiting on auth endpoints\n7. **CSRF Protection**: Use state parameter (already implemented)\n8. **Account Linking**: Require email verification before linking\n\n## Part 10: Monitoring and Analytics\n\n### Enable Event Logging\n\n1. Go to Realm Settings → Events\n2. Enable \"Save Events\"\n3. Enable \"Save Admin Events\"\n4. Set \"Expiration\": 30 days\n\n### View Login Events\n\n1. Go to \"Events\"\n2. Filter by:\n   - Event Type: `LOGIN`\n   - Client: `agilestest-web`\n   - Identity Provider: `google` or `github`\n\n### Export Events\n\n```bash\n# Export events to JSON\ncurl -X GET http://localhost:8080/admin/realms/agilestest/events \\\n  -H \"Authorization: Bearer $TOKEN\" | jq > events.json\n```\n\n## References\n\n- [Google OAuth 2.0 Documentation](https://developers.google.com/identity/protocols/oauth2)\n- [GitHub OAuth Documentation](https://docs.github.com/en/developers/apps/building-oauth-apps)\n- [Keycloak Social Login Documentation](https://www.keycloak.org/docs/latest/server_admin/#_social_identity_providers)\n- [Keycloak Identity Provider Configuration](https://www.keycloak.org/docs/latest/server_admin/#_identity_provider_config)\n- [OpenID Connect Specification](https://openid.net/connect/)\n\n## Checklist\n\n- [ ] Google Cloud Project created\n- [ ] Google+ API enabled\n- [ ] Google OAuth credentials obtained\n- [ ] GitHub OAuth app registered\n- [ ] GitHub credentials obtained\n- [ ] Google configured in Keycloak\n- [ ] GitHub configured in Keycloak\n- [ ] Google login tested\n- [ ] GitHub login tested\n- [ ] User creation verified\n- [ ] Production URIs configured\n- [ ] Event logging enabled\n- [ ] Security best practices implemented\n
